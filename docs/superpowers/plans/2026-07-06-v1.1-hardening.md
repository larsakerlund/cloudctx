# cloudctx v1.1 Hardening Implementation Plan

> Executed inline in the authoring session (full review context in hand), one
> commit per task, TDD per task: failing test тЖТ minimal change тЖТ green suite.
> Spec: `docs/superpowers/specs/2026-07-06-v1.1-hardening-design.md`.

**Goal:** Close the 2026-07-06 review findings: guard coverage, registry type
safety, install idempotency, status timeouts, `delete`/`show`/`list -v`,
`--version`, completion, post-login verification.

## Tasks

### Task 1: Registry drops non-string values
- `load_registry` keeps only `str` values per entry; warning to stderr for
  anything else. Never coerce (a bool must not become subscription "True").
- Test `test_load_registry_drops_non_string_values`: write a registry with
  `flag = true` beside a valid string key; assert all loaded values are str
  and `_env` on that context exits cleanly (works on both tomllib and the
  3.9 fallback, which parses everything as strings anyway).

### Task 2: `_probe_json` with timeout in status
- New helper `_probe_json(argv, timeout=PROBE_TIMEOUT, env=None) ->
  (status, data)`; `cmd_status` uses it for az and aws probes; timeout
  renders `(timed out)` lines. `PROBE_TIMEOUT = 10`.
- Tests: helper returns `("timeout", None)` for a `sleep` stub with a tiny
  timeout; `("error", None)` for exit 1; `("ok", dict)` for JSON output.
  Existing `test_status_*` stay green.

### Task 3: install idempotency + `--version`
- `cmd_install --write`: if rc already contains the exact source line, print
  "already installed" and return 0.
- `__version__ = "1.1.0"`; `--version` argparse action on the root parser.
- Tests: double `--write` leaves exactly one source line; `cloudctx --version`
  (subprocess) prints `cloudctx 1.1.0`.

### Task 4: guard segment splitting (both shims)
- `_cctx_guard`: replace `&&`/`||`/`;`/`|`/`&` with newlines, loop segments,
  run the existing wrapper-skip + az/aws check per segment; add `-*` to the
  wrapper-skip case.
- Shell tests (zsh + bash via the existing subclass): `cd x && az login`,
  `echo hi | aws s3 ls`, `true; az login`, `stdbuf -oL az login` all warn
  with no context; `ls && echo done` stays silent.

### Task 5: `cloudctx delete`
- `cmd_delete(name, --force, --keep-store)` per spec ┬з2.5; EOF/anything but
  y/yes aborts with exit 1; store removed via `shutil.rmtree`; profiles
  regenerated (OSError tolerated); warn if `$CLOUDCTX_CONTEXT == name`.
- Tests: force-delete removes entry+store and regenerates profiles;
  `--keep-store` keeps the dir; unknown name errors; stdin "n" aborts and
  changes nothing; stdin "y" deletes.

### Task 6: `list -v`, `show`, `_names`
- `cmd_list -v`: aligned columns (marker, name, display, azure sub|tenant,
  aws profile). `cmd_show <name>`: entry keys sorted + store paths with
  exists-markers. `_names`: bare sorted names.
- Tests: verbose list contains display + subscription; show prints paths;
  `_names` prints exactly the names.

### Task 7: completion in shims
- zsh: `_cloudctx` function + `compdef _cloudctx cloudctx` guarded by
  `(( $+functions[compdef] ))`. bash: `_cloudctx_complete` + `complete -F`.
  Both complete subcommands at position 1 and `_names` output at position 2
  for `use login exec open claude delete show`.
- Tests: zsh тАФ stub `compdef` before sourcing, assert `_cloudctx` defined and
  registered; bash тАФ `complete -p cloudctx` names `_cloudctx_complete`.

### Task 8: post-login verification
- After non-dry-run Azure login: `_probe_json(["az","account","show","-o",
  "json"], env=ctx_env)`; print `logged in: <name> (<id>) tenant=<tenantId>`;
  GUID-shaped registry tenant mismatch тЖТ stderr warning. Probe failure is
  non-fatal (login already succeeded).
- Test: stub `az` returning mismatching tenantId JSON тЖТ warning on stderr;
  matching tenant тЖТ no warning.

### Task 9: README + finish
- README: `delete`/`show`/`list -v`/`--version` in usage, completion note.
- Full suite + isolation proof green тЖТ merge to main, push (user's standing
  preference), delete branch.
