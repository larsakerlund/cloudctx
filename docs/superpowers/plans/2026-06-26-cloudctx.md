# cloudctx Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A zero-dependency tool that pins each iTerm2 window to one named cloud context, so Azure (and, later, AWS) logins in one window never affect another.

**Architecture:** A single executable Python CLI (`cloudctx`, argparse, stdlib only) owns all logic; a thin sourced `ctx` shell function (`zsh` + `bash`) mutates the parent shell by `eval`-ing `cloudctx _env <name>` and then running `cloudctx _decorate <name>` for iTerm2 visuals. Per-context credential stores live under `~/.cloudctx/<name>/` and are selected purely through environment variables.

**Tech Stack:** Python 3.9+ stdlib (argparse, json, base64, subprocess, shutil, pathlib, optional `tomllib`), zsh/bash, iTerm2 escape codes + Dynamic Profiles + AppleScript, Starship `env_var` module. Tests: stdlib `unittest` (no pip install) + zsh/bash subprocess tests.

## Global Constraints

- **Zero runtime dependencies.** Stdlib only. No venv, no pip packages. Installs by copying the `cloudctx` file onto `PATH`.
- **Python floor 3.9.** `tomllib` is 3.11+ and read-only; reading uses `tomllib` when importable, else a built-in mini-parser; writing is always hand-rolled.
- **Tests use stdlib `unittest`** run via `python3 -m unittest`, loading the extensionless `cloudctx` file via `importlib.util`.
- **Injection safety:** every value `_env` emits is single-quote-escaped via the POSIX `'\''` idiom; the registry is treated as untrusted input to the `eval`.
- **Non-destructive:** never write `~/.azure`, `~/.aws`, or the `default` AWS profile. Never auto-migrate.
- **v1 is Azure-only for auth automation.** AWS fields are parsed and exported when present, but `login`/`new` only automate Azure; AWS auth prints a clear "not yet implemented; install AWS CLI" stub.
- **Registry root:** `~/.cloudctx`, overridable via `$CLOUDCTX_HOME` (for tests).
- **Verified facts are in `docs/superpowers/specs/2026-06-26-cloudctx-design.md` §8 sources.** Escape codes and CLI semantics below are copied from that verified research — do not "correct" them from memory.

### Verified technical reference (do not re-derive)

- **Badge:** `printf '\e]1337;SetBadgeFormat=%s\a' "$(printf %s "<text>" | base64)"` — base64 **required**. Clear: empty payload.
- **Title:** `printf '\e]0;%s\a' "<text>"` (OSC 0 sets tab+window). Plain text, no base64.
- **Tab color:** `printf '\e]1337;SetColors=tab=%s\a' "<rrggbb>"` — **hex, no `#`**. Reset: `tab=default`.
- **Dynamic profile:** `{"Profiles":[{"Name":..,"Guid":<stable>,"Initial Text":"ctx use <name>","Use Tab Color":true,"Tab Color":{"Red Component":f,"Green Component":f,"Blue Component":f,"Alpha Component":1,"Color Space":"sRGB"}}]}` — components are floats `0.0–1.0` (value/255).
- **Open window:** `osascript` argv: `-e 'tell application "iTerm2"' -e 'set w to (create window with default profile)' -e 'tell current session of w' -e 'write text "<cmd>"' -e 'end tell' -e 'end tell'`; escape `\`→`\\`, `"`→`\"` inside the AppleScript literal; invoke osascript via argv array (no shell).
- **zsh preexec:** `autoload -Uz add-zsh-hook; add-zsh-hook preexec cloudctx_preexec`; function arg `$1` = typed line; split with `${(z)1}`, test `${words[1]}` ∈ {az,aws}.
- **zsh prompt:** `setopt PROMPT_SUBST; PROMPT='${CLOUDCTX_CONTEXT:+%F{cyan}[$CLOUDCTX_CONTEXT]%f }'"$PROMPT"` (single quotes).
- **bash preexec:** `trap 'cloudctx_debug' DEBUG` reading `$BASH_COMMAND`, gated by a once-per-prompt flag reset in `PROMPT_COMMAND`.
- **Starship:** `[env_var.CLOUDCTX_CONTEXT]` module + add `${env_var.CLOUDCTX_CONTEXT}` to top-level `format`.
- **Azure:** `AZURE_CONFIG_DIR` (default `~/.azure`) holds token cache + `azureProfile.json`. `az login --tenant <id> [--use-device-code]`; `az account set --subscription <id-or-name>`; `az account show` fields: `name`, `id`, `tenantId`, `user.name`.
- **AWS env:** `AWS_CONFIG_FILE`, `AWS_SHARED_CREDENTIALS_FILE`, `AWS_PROFILE`. `~/.aws/sso/cache` cannot be relocated — leave it shared (safe; keyed per SSO start URL).

## File Structure

| File | Responsibility |
|---|---|
| `cloudctx` | The entire CLI. Sections: registry/TOML, color helpers, env emission, decorate emission, commands (`list`/`new`/`login`/`status`/`exec`/`open`/`claude`/`gen-profiles`/`install`), argparse dispatch. |

Single-file CLI is a deliberate trade-off: distribution = copy one file, and `ctx use` pays no import cost beyond stdlib. The file is organized into clearly commented sections so it stays navigable.

**Files:** Create `shell/ctx.zsh`, `shell/ctx.bash`; Test `tests/test_shell.py`.

**Interfaces — Consumes:** `cloudctx _env`, `cloudctx _decorate`. **Produces:** a sourced `ctx` function.

`shell/ctx.zsh` core:

```zsh
ctx() {
  emulate -L zsh
  case "$1" in
    use)
      [[ -z "$2" ]] && { echo "usage: ctx use <name>" >&2; return 1; }
      local env; env="$(command cloudctx _env "$2")" || return 1
      eval "$env"
      command cloudctx _decorate "$2"
      ;;
    clear)
      eval "$(command cloudctx _env --clear)"
      command cloudctx _decorate --clear
      ;;
    *) command cloudctx "$@" ;;
  esac
}
```

(`bash` variant: `local` works; replace `emulate -L zsh` with nothing; identical otherwise.)
